# NetTool - 网络工具 × AI 集成平台

NetTool 是一个**网络工具与 AI 深度集成**的综合平台，致力于将日常网络管理工作中常见的工具、操作和流程智能化，让网络工程师能够用自然语言驱动网络设备，用 AI 辅助设计和诊断网络拓扑。

> 未来的网络运维，不是人找工具，而是工具听懂人话。

## 平台架构

```
net_tool/
├── netops/      # 网络拓扑编辑器（已上线）
├── tools/       # 共享基础设施和工具库
└── ...
```

| 目录 | 说明 |
|------|------|
| `netops/` | 第一个已上线工具：网络拓扑编辑器 + AI 助手 + 终端 |
| `tools/` | 各工具共享的基础设施和公共模块 |

## 核心设计理念

- **AI Native**：AI 不是附加功能，而是渗透到每个工具的核心操作中
- **自然语言驱动**：用户说"加一台路由器连到核心交换"，AI 自动完成拓扑变更
- **工具可扩展**：未来每个网络工具都可以是平台的一个模块，共享 AI 能力
- **本地优先**：数据存储在本地，不强依赖云端

## NetOps - 网络拓扑编辑器

NetOps 是 NetTool 平台发布的第一个工具，定位是**交互式网络拓扑编辑器 + AI 助手 + 设备终端**的一体化工作台。

### 功能概览

| 功能 | 说明 |
|------|------|
| 🖱️ 拓扑编辑 | 可视化拖拽式编辑，支持路由器/交换机/防火墙/服务器等设备 |
| 🤖 AI 助手 | 内置 AI 理解拓扑，自然语言添加/修改/删除设备和连线 |
| 💻 设备终端 | 浏览器内直连网络设备，支持 Telnet/SSH |
| 💾 多项目管理 | 每个项目独立存储，支持变更追溯 |

### 快速开始

```bash
cd netops
bash start.sh
```

访问地址：`http://localhost:9000`

## 技术栈

- **前端**：Cytoscape.js、Cytoscape.js npm extensions
- **后端**：Python 3（自定义 HTTP 服务器）
- **终端**：xterm.js + Python PTY
- **AI**：MiniMax M2.5-highspeed（内置，无需外接服务）
